************** COMMITS **************
commits
$ git status - para saber o estado dos codigos, qual esta modificado e quais ja foram enviado 
$ git add "nome do arquivo" - para adicionar o arquivo especifico para a fila do commit
$ git add . - para adicionar todos os arquivos modificados para a fila do commit
$ git commit -m "mensagem que da para aquele commit" - para dar o nome daquela commit que voce esta enviando para o github

$git push - onde voce ira puxar os arquivos para aquela branch
//OBS: as vezes o programa vai pedir que voce loga no github ou se caso voce estiver conectado ele vai pedir somente o login e a senha



************** criando uma branch **************
branch - são pastas onde voce vai trabalhar o seu codigo
para visualizar as branch
$ git branch
// logo apos isso aparecerá as branchs(pastas) feitas e a pasta usada no momento ( geralmente esta marcado com uma cor )

para criar um branch
$ git branch nome_da_branch(pasta)

para a sua branch criada seja a sua principal
$ git checkout nome_da_branch

OBS: voce pode usar um código só para criar e fazer com que sua branch seja a principal utilizando esse comando $ git checkout -b nome_da_branch

************** unindo os codigos de branch diferentes (git merge) **************
// é aconselhavel olhar qual branch esta para mandar esse codigo "USE O $ git branch"
$ git checkout master (a branch onde voce quer colocar os codigos)
$ git merge staging ( a branch onde vc fez as modificações sem alterar o codigo ja em funcionamento ) 
 /////// OBS: antes de usar o git merge nome_da_branch sempre usa um git pull pois se vc fez alguma alteração ou o usuario fez alteração la pode dar problema na hora da modificação SEGUE A ETAPA:

1- $git pull da branch principal
2- gerar uma nova branch a partir da branch principal
3- trabalhar e adicionar novas funcionalidades na nova branch que criou.
4- finalizar o trabalho na branch temporária
5- $ git checkout na branch principal
6- $ git pull
5- Mergiar(unir) o código da branch temporário com a branch principal(depois de testar)
6- $ git push da branch principal



************** unindo os codigos de branch atravez do $ git pull request (merge democratico) **************
na branch que voce esta modificando para poder enviar para a branch MASTER
1- vc faz as modificações e utiliza o $ git add . 
2- e faz o git commit -m "mensagem"
3- utiliza o git push

apos fazer isso seus arquivos esta na nuvem e é só voce entrar no github e nos arquivos modificado na sua branch vc usa o merge da branch que vc ta para a branch MASTER e solicitar um pull request
voce ira para uma pagina onde voce falara o que voce mudou e embaixo explicara os detalhes

o request seria uma solicitação para os outros desenvolvedores possam ver e o chefe aprovar essa solicitação e mergiar na branch master


*************comando GIT IGNORE***************
para voce não quer enviar alguns arquivos sobe para o controle de versão

no $ git status ira aparecer 
para usar o git ignore  voce precisa ir no terminal do git

lá voce usa esse comando: $git touch .gitignore

aparecera na pasta  onde trabalhara no seu projeto o arquivo .gitignore basta acessar no bloco de notas

no bloco de notas voce digitara o nome do arquivo e o formato dele ex:
se for pasta: nome da pasta/
se for foto: foto.png
arquivo: nome do arquivo.tipo do arquivo
